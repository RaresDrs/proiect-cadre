### Accent Reserved For (explicit list — never "all interactive elements")

1. Primary CTA button background (nav "Get Started" + Hero primary CTA)
2. Hero CTA primary button hover state
3. Active/current nav link underline indicator
4. Pricing Free tier badge border
5. Email capture "Notify Me" submit button
6. Focus ring on all interactive elements (2px solid, 2px offset)
7. PWA theme_color in manifest.json

Accent is NOT used on: secondary buttons, FAQ toggles, feature card icons (use #1D1D1F / #F5F5F7), footer links, or general hover states.

## Animation Contract

### Hero Section (Framer Motion — lazy loaded)

| Element | Animation | Duration | Delay | Easing |
|---------|-----------|----------|-------|--------|
| Headline | fade-in + translateY(20px → 0) | 500ms | 0ms | spring (stiffness 100, damping 20) |
| Sub-headline | fade-in + translateY(20px → 0) | 500ms | 150ms | spring (stiffness 100, damping 20) |
| CTA buttons | fade-in + translateY(16px → 0) | 400ms | 300ms | spring (stiffness 120, damping 22) |
| Structural diagram | fade-in + scale(0.95 → 1.0) | 600ms | 200ms | spring (stiffness 80, damping 18) |

### All Other Sections (tw-animate-css + IntersectionObserver)

| Class | Trigger | Duration |
|-------|---------|----------|
| animate-fade-in-up | element enters viewport (threshold 0.15) | 300ms |
| animate-fade-in | section headings | 250ms |

Rules:
- Max 2 elements animating simultaneously in any viewport
- transform and opacity only — no layout-affecting properties
- ease-out on all entrance animations
- `prefers-reduced-motion: reduce` → all animations disabled, elements render at final state immediately

## Dark Mode Implementation

| Concern | Implementation |
|---------|----------------|
| Class strategy | `.dark` on `<html>` element |
| Persistence | `localStorage` key: `structcalc-theme` |
| First visit | Auto-detect via `window.matchMedia('(prefers-color-scheme: dark)')` |
| Flash prevention | Blocking `<script>` in `<head>` before any CSS loads — reads localStorage, sets class synchronously |
| Toggle | lucide `Sun` (light mode active) / `Moon` (dark mode active) icon button in nav |
| CSS variable extension | Extend `:root` and `.dark` blocks in `frontend/src/index.css` — do NOT rewrite existing structure |

| Scope | All user-visible strings: nav links, section headings, body copy, CTA labels, FAQ content, footer |

## Accessibility Contract

| Concern | Requirement |
|---------|-------------|
| Color contrast | Body text on background: ≥ 4.5:1 (WCAG AA). Verified: #1D1D1F on #F5F5F7 = 16.1:1. Accent #2563EB on white = 4.56:1 (AA pass). |
| Focus rings | 2px solid #2563EB, 2px offset — all interactive elements |
| Touch targets | Minimum 44x44px for all buttons, toggles, nav links |
| Reduced motion | All Framer Motion + tw-animate-css animations disabled when prefers-reduced-motion: reduce |
| Semantic HTML | Nav uses `<nav>`, sections use `<section id="...">`, headings use correct h1/h2/h3 hierarchy |
| h1 rule | Exactly one `<h1>` per page — Hero headline only |
| ARIA | FAQ accordion: aria-expanded on trigger, aria-controls pointing to content panel |
| ARIA (icon-only) | DarkModeToggle is icon-only — must include `aria-label="Comută modul întunecat"` (RO) / `aria-label="Toggle dark mode"` (EN), updated dynamically on language toggle |
| Language | `<html lang="ro">` default, updated to `lang="en"` on toggle |
| Skip link | `<a href="#features" class="sr-only focus:not-sr-only">` at top of `<body>` |
